Goal
- Build a library implementing the README algorithm: Kafka ingest -> DB READY rows -> ordered per-key claim/process -> retries/STOPPED -> heartbeat -> orphan reclaim.

Algorithm (from README)
1) Consumer persists Kafka records to `messages` with deterministic `uuid` and `status=READY`.
2) Worker claims batch atomically with per-key ordering guard:
   - eligible: READY or FAILED with retry_count < maxRetries
   - blocked if older row of same key is READY/FAILED/PROCESSING
   - blocked if same key already has PROCESSING row
   - claimed rows become PROCESSING + container_id
3) Business handler runs per claimed row.
4) SUCCESS => PROCESSED; RETRY => increment retry_count then READY or STOPPED.
5) Worker heartbeats written to `container_heartbeats`.
6) Orphan reclaimer resets stale/dead-worker PROCESSING rows to READY.
